포인터(pointer)
============
## 1. 포인터의 정의
변수의 **메모리 공간의 주소**를 가리키는 변수
```c
int num = 10;
 int* p = &num;  //p는 num의 주소를 저장하게 된다.
```
  

## 2. 포인터 변수의 선언
`자료형* 변수명;`형태로 선언한다.

```c
  int* p1;
  double* p2;
  char* p3;
```

---
## 3. 주소 연산자(&)와 역참조 연산자(*)

- **`&` (주소 연산자)** : 변수 앞에 붙이면 "그 변수의 주소"를 가져온다.
- **`*` (역참조 연산자)** : 포인터 앞에 붙이면 "그 포인터가 가리키는 곳의 값"을 가져온다. (선언할 때 쓰는 `*`와는 다른 의미)

```c
int num = 10;
int* p = &num;    
printf("%d", *p);  // * : p가 가리키는 곳의 값(10)을 출력한다.
*p = 20;           // p가 가리키는 곳(num)의 값을 20으로 바꿈, p = 20 이렇게 바꿀 수는 X
```

---

## 4. 포인터 변수의 크기

포인터가 가리키는 자료형이 무엇이든, **포인터 변수 자체의 크기는 시스템에 따라 항상 동일**하다.
→ 포인터는 결국 "주소값"(숫자)만 저장

---

## 5. 포인터와 배열의 관계

배열 이름은 사실 **배열의 첫 번째 요소의 주소**와 같은 값으로 취급된다.

```c
int arr[5] = {1,2,3,4,5};
int* p = arr;        // arr은 &arr[0]과 같은 의미
printf("%d", *p);      // 1 (=arr[0])
printf("%d", *(p+1));  // 2 (= arr[1])
```

단, `arr` 자체는 상수라서 `arr = p2;`처럼 다른 주소를 대입할 수 없다. 반면 포인터 변수는 가리키는 곳을 자유롭게 바꿀 수 있다.

---

## 6. 포인터 산술 연산

- **포인터 + 정수** : 몇 칸 뒤의 주소로 이동 (그 타입의 크기만큼 실제 주소가 증가)
- **포인터 - 포인터** (같은 타입) : 두 주소 사이에 몇 개의 요소가 있는지 계산
- **포인터끼리 덧셈**, **곱셈/나눗셈/나머지 연산** : 불가능

```c
int arr[5];
int* p1 = &arr[0];
int* p2 = &arr[3];
p2 - p1;   // 3 (요소 3개 차이)
p1 + 2;    // &arr[2]와 같은 주소
```

---

## 7. 포인터와 함수 — call by reference

함수에 변수를 그냥 넘기면(**call by value**) 값만 복사되어, 함수 안에서 값을 바꿔도 원본은 바뀌지 않는다.
하지만 **주소를 넘기면(call by reference)**, 함수 안에서 원본 값을 직접 바꿀 수 있다.

```c
void changeValue(int* p) {
    *p = 100;
}

int main() {
    int num = 10;
    changeValue(&num);
    printf("%d", num); // 100
}
```

---

## 8. 배열을 함수의 인자로 전달하기

배열은 함수에 넘길 때 자동으로 **시작 주소만** 전달된다(call by reference와 비슷). 그래서 함수 안에서 배열 값을 바꾸면 원본 배열도 바뀐다.

```c
void Function(int* array, int n) {
    array[0] = 99;
}

int main() {
    int arr[5] = {1,2,3,4,5};
    Function(arr, 5);
    printf("%d", arr[0]); // 99
}
```

함수는 시작 주소만 받으므로, **배열의 길이(개수)도 매개변수로 같이 넘겨줘야 범위를 벗어난 접근을 막을 수 있다.**

---

## 9. sizeof 연산자와 배열 크기 구하기

`sizeof(배열)`은 배열 전체가 차지하는 바이트 크기를 알려준다.

```
배열 전체 크기 ÷ 요소 1개 크기 = 배열의 개수
```

```c
int arr[5];
sizeof(arr) / sizeof(arr[0]); // 5
```

이건 **배열 자체**를 갖고 있을 때만 가능하다. 함수 안에서 포인터로 받은 경우엔 시작 주소밖에 없어서 이 계산이 통하지 않는다 (그래서 8번에서 길이를 따로 넘겨야 했던 것).

---

## 10. 2차원 배열과 포인터

2차원 배열도 결국 메모리상에서는 한 줄로 쭉 이어진 1차원 배열이다. 함수에 넘길 때 여러 표현 방법이 있다.

```c
void myPrint(int a[4][3], int row, int col);
void myPrint(int a[][3], int row, int col);
void myPrint(int (*a)[3], int row, int col);
```

어떤 표현이든 **열의 개수(한 줄에 몇 칸인지)**는 꼭 알려줘야 정확한 위치를 계산할 수 있다.

---

## 11. 포인터와 문자열

```c
char s1[] = "Hello";   // 배열 → 내용 수정 가능
char* s2 = "Hello";    // 포인터 → 내용 수정 불가(상수 영역을 가리킴), 대신 가리키는 곳 자체는 바꿀 수 있음
```

- 문자열은 끝에 자동으로 `'\0'`
